Internal Manual: The Architect’s Vault
1.1 The Definition of Done (Version 1.0)
To prevent Scope Creep and ensure this project reaches the finish line, Version 1.0 is strictly defined as an Internal Employee Utility. We will not move to Phase 2 (Client-Facing) until the following criteria are met 100%.
I. The "Agnostic" Pipeline Requirements
The engine must successfully ingest and "normalize" the following without crashing the browser:
Formats: JPEG, PNG, HEIC (iPhone), and Single-Page TIFF.
Normalization: Every input must be converted to a standard Canvas object.
Rotation: User must be able to rotate any page in 90° increments.
Optimization: The "Stapler" must output a PDF where images are downsampled to 150 DPI.
II. The "Tabletop" UI (v1.0)
The interface is considered "Done" when an employee can perform these 4 actions:
Drop Zone: Drag 10+ files of mixed formats into the window at once.
Visual Sort: Drag a thumbnail to change its position in the final PDF sequence.
Delete: Remove a "bad" page (e.g., a blurry photo) before the merge.
Export: A single click generates the PDF and triggers a browser download.
III. Performance Benchmarks
Memory Management: The app must be able to process 20 high-res mobile photos (approx. 5MB each) on a machine with 8GB of RAM without a "Page Unresponsive" error.
Speed: The final PDF generation must take less than 10 seconds once the "Export" button is clicked.
1.2 The "Internal Guardrails" (Anti-Distraction)
To stay focused, the following are EXPLICITLY EXCLUDED from Version 1.0:
No Cloud Saving: Files are download-only. No Supabase storage integration yet.
No AI/OCR: No automatic text reading or form-filling.
No Mobile-First UI: While it should work on a tablet, the v1.0 design is optimized for a desktop browser (Employee Workstation).
No User Accounts: Version 1.0 will use a "Single License Key" or be restricted by IP/Domain to keep Auth logic simple.
1.3 Personal Success Milestone (The Reward)
The "Proof of Life" Test: Successfully "staple" a sideways HEIC and a vertical PNG into a single, upright, 2-page PDF.
The Reward: [INSERT YOUR REWARD HERE - e.g., "The new mechanical keyboard" or "A weekend completely off-grid."]
Why this works for you:
By documenting these exclusions, you give yourself permission to ignore the "cool ideas" that pop up during coding. If you think, "Oh, I should add a feature to email the PDF," you look at this manual and say, "No, that’s Phase 2."
