Internal Manual: Section 1.2 — Technical Architecture
I. The Monorepo Structure (pnpm + Turborepo)
We use a workspace-based approach to separate the "Engine" (Logic) from the "Skin" (UI).

II. The "Agnostic Pipeline" Flow (Internal Logic)
To ensure Version 1.0 is stable, we follow a strict data flow. No raw files should ever touch the "Stapler" (PDF Generator) directly. They must go through the Normalizer.
Ingestion: User drops a file (HEIC, TIFF, PNG, etc.).
Normalization: A Worker thread converts the file into a standardized Bitmap/Canvas object.
The State: The app stores a list of these objects in the "Tabletop" state.
Transformation: Any user action (Rotate/Reorder) updates the metadata of the object in the state.
Composition: On export, the pdf-engine takes the objects, applies the transformations, and "paints" them into the final PDF.
III. Dependency Guardrails
To keep the project "Completion-Proof," we strictly limit the libraries we use. If a library is added, it must be documented here.
PDF Generation: pdf-lib (Low-level, client-side manipulation).
Image Handling: heic2any (for Apple formats) and canvas API (for native browser scaling).
UI Framework: Lit (to ensure the components can be moved to any framework in Phase 2).
IV. Technical "Debt" Log (The "Do Not Fix Yet" List)
To reach the finish line, we are intentionally allowing the following "Controlled Debt" in v1.0:
Local State Only: If the user refreshes the page, they lose their progress. (Fix in Phase 2 with Supabase/LocalDB).
Synchronous Processing: We will start with a single-thread model. If 50+ images cause lag, we move to Web Workers only after the first successful demo.
Minimalist Styling: Use standard system fonts and a CSS framework like Tailwind to avoid hours of custom CSS work.
Why this structure prevents "The Wall":
If you get bored with the UI, you can spend a day working on the "Engine" package. If the math in the Engine gets too heavy, you can switch to the "Manual" or the "UI Library." You have different "rooms" to work in, which keeps your brain engaged without losing overall momentum.
